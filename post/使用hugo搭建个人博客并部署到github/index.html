使用hugo搭建个人博客并部署到github

2021/08/13

1. Hugo安装

本人操作系统为windows10 64位。

到Hugo releases页面下载hugo_xxx_Windows-64bit.zip，解压到一个安全目录(D:/hugo)，然后将该目录添加到环境变量path中，在cmd运行version，正确展示版本号则安装正确。

2.Hugo环境配置

在path添加：D:\hugo\

3.生成站点

 hugo new site your_site_name

your_site_name可以任意取名。

站点目录结构：

  ▸ archetypes/ 
  ▸ content/     #静态页面目录
  ▸ data/
  ▸ layouts/
  ▸ resourse/
  ▸ static/      #静态文件目录
  ▸ themes/      #主题目录
  - config.toml  #配置文件

4.创建页面

$ hugo new about.md

about.md 自动生成到了 content/about.md 。

若md文件内有draft=false，在部署到github时应该改为true。

内容是 Markdown 格式的，+++ 之间的内容是 TOML 格式的。

5.创建文章

$ hugo new post/firstBlog.md

6.布置博客主题

在hugo主题中挑选一个主题，将其下载到站点的themes目录下。可直接下载或者使用git。

本人使用anatole主题（主题简洁；并且有config.toml文件而不是yaml文件，exampleSite文件可供参考，布置比较简单)。

$ cd themes
$ git clone https://github.com/lxndrblz/anatole.git

7.测试主题

在你的站点根目录执行 Hugo 命令进行调试：

$ hugo server --theme=anatole --buildDrafts

浏览器里打开： http://localhost:1313

8.建立仓库

我的ssh路径：https://github.com/zhengshi119/zhengshi119.github.io.git

9.修改配置文件config.toml

baseURL = "https://zhengshi119.github.io"
languageCode = "zh-cn"
DefaultContentLanguage = "zh"
title = "Website of Zhengshi119"
theme = "anatole"

注意修改baseURL（你的githubPage)、theme（你下载的主题名称)，其他的根据个人自行修改。

10.生成public目录

执行hudo命令,生成public目录:
hugo

所有静态页面都会生成到 public 目录。

11.部署到github上

$ cd public
$ git init
$ git remote add origin https://github.com/zhengshi119/zhengshi119.github.io.git
$ git add -A
$ git commit -m "first commit"
$ git push -u origin master

这里输入的是你github上的邮箱，之后一个输入你github的密码。

注意：在github远程创建仓库后, 利用gitbash进行提交本地文件的时候出现错误时，可以试试：

1: 进行push前先将远程仓库pull到本地仓库
$ git pull origin master    #git pull --rebase origin master
$ git push -u origin master
 
2: 强制push本地仓库到远程 (这种情况不会进行merge, 强制push后远程文件可能会丢失 不建议使用此方法)
$ git push -u origin master -f
 
3: 避开解决冲突, 将本地文件暂时提交到远程新建的分支中
$ git branch [name]
# 创建完branch后, 再进行push
$ git push -u origin [name]

我使用的是第二个的强制push。

12.在github上修改分支

因为站点是部署在master分支上的，需要修改到master分支，才能显示你修改的界面。

在Setting→Pages：

最后在浏览器上登录：https://zhengshi119.github.io
